问题

引入 @dnd-kit 实现 Project 列表拖拽排序后，默认 PointerSensor 在 pointerdown 瞬间就激活拖拽跟踪，调用 preventDefault() 吞掉了所有子元素的 click 事件。导致 ProjectItem 内的 toggle 折叠/展开、按钮操作、右键菜单全部失效。

根因

  • DndContext 未配置 sensors，走默认的 PointerSensor（零距离激活）
  • ProjectItem / ConnectionProjectCard 将 useSortable 返回的 {...listeners} 展开在最外层 div，覆盖了所有子元素

修复

为三处 DndContext 添加带 activationConstraint: { distance: 5 } 的 PointerSensor + KeyboardSensor：

  • src/features/project/components/ProjectsPanel.tsx — 本地项目列表拖拽
  • src/features/connection/components/RemoteItems.tsx — WSL 项目列表拖拽
  • src/features/connection/components/RemoteItems.tsx — SSH 远程项目列表拖拽

指针移动 5px 以上才启动拖拽，正常点击（toggle、按钮、右键菜单）全部恢复正常。
