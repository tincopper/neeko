# 会话历史标题与摘要规范化

## Goal

将会话历史列表中的标题和摘要（preview）规范化，对齐 orca 参考实现的净化管线，确保展示给用户的文字无 ANSI 转义、无 harness 注入噪声、无隐藏系统标签、统一省略号格式，并抑制无意义的系统指令前缀。

## Requirements

### R1: 标题净化管线 (`normalize_session_text`)

1. 剥离 ANSI 转义序列
2. Unicode 空白折叠（多种空白 → 单个空格）
3. 移除隐藏块标签对及其内容（`<system-reminder>`、`<codex_internal_context>`、`<goal_context>`）
4. 抑制前缀检测：以 `# AGENTS.md instructions` 或 `<INSTRUCTIONS>` 开头 → 返回 `None`
5. 剥离 harness 句子级前缀（`"Caveat: The messages below were generated by the user while running local commands"`、`"This session is being continued from a previous conversation"`）
6. 字符边界截断（Title ≤96 chars, Preview ≤220 chars）并追加 `...`（三连点，非 `…`）
7. UTF-16 代理对安全：截断不切开高代理对

### R2: 三优先级标题解析 (`resolve_title`)

- **P2**: AI 生成标题（适配器提供，经 `normalize_session_text` 净化）
- **P3**: 首条用户消息（经 `normalize_session_text` 净化）
- **Fallback**: `<agent_id> <native_session_id 前 8 位>`

### R3: 预览文本构建 (`build_preview_messages`)

- 从 `recent_messages` 环形缓冲（role, text 对）构建
- 排除 harness 注入噪声消息（`is_harness_injected_user_turn`：12+ 条已知前缀）
- 排除空白消息
- 保留最近 ≤5 条真实消息（`PREVIEW_MESSAGE_LIMIT`）
- 每条消息经 `normalize_session_text` 净化后拼为 `Role: text` 多行

### R4: 适配器数据源

7 个 AgentSessionAdapter 各自从原生存储格式读取最近消息对：

| 适配器 | 路径/格式 |
|--------|----------|
| ClaudeCode | `~/.claude/sessions/*.jsonl` (树形) |
| Codex | `~/.codex/sessions/*.jsonl` (树形) |
| Pi | `~/.pi/sessions/*.jsonl` (树形) |
| Gemini | `~/.gemini/tmp/*/chats/*.json` |
| Qoder | `~/.qodercli/projects/*/*.jsonl` (平面) |
| CodeBuddy | `~/.codebuddy/sessions/*.json` |
| OpenCode | `~/.local/share/opencode/opencode.db` (SQLite) |
